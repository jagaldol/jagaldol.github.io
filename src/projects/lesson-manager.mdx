부산대/경성대 2개 지점을 운영 중인 피아노 학원, 피아노투게더의 레슨 관리 시스템을 웹 어플리케이션으로 제작하였습니다.
친구가 다니던 피아노 학원의 레슨 관리 시스템이 너무 느리고, 불편하며, 디자인이 이쁘지 않았습니다. 학원 대표님께 저희가 직접 더 나은 시스템과 디자인으로 만들 수 있다고 설득하였고, 일을 수주받아 친구와 2인 팀을 결성하여 개발을 진행하였습니다.

레슨 매니저는 학원의 레슨 시간표를 관리할 수 있는 웹 어플리케이션입니다. 수강생은 자신의 아이디를 사용하여 레슨 시간표 조회/ 레슨 예약/ 레슨 취소 등을 직접 할 수 있습니다. 관리자(매니저/강사)는 추가로 수강생의 레슨을 대신 예약해줄 수 있으며, 수강생 관리(등록/탈퇴, 검색, 상세정보 조회, 결재 내역 조회 및 환불, 안내 문자 전송 등), 강사 관리(등록/퇴사 처리, 상세정보 조회, 담당학생 조회, 출근 시간표 관리 등), 결제 정보 등록이 가능합니다. 즉, 수강생에게는 웹을 통해 쉽게 학원의 레슨을 예약/조회 할 수 있고, 매니저에게는 학원의 강사, 수강생, 결제 등 전체적인 학원 시스템을 일원화하여 관리할 수 있는 프로그램입니다.

해당 프로젝트는 2인 팀으로 둘 다 풀스택 개발자로 참여하였습니다. 먼저 백엔드 쪽에서 직접 맡은 부분은 database 설계, 관리자 전용 api 설계 및 구현, 문자 전송입니다. 프론트 쪽에서는 수강생 관리 페이지, 강사 관리 페이지, 모바일 대응 반응형 화면 구성을 맡았습니다.

2개월 간의 개발 기간 이후, 실제 학원의 서비스로 연결시켰고 2주일 동안 자잘한 버그 수정(ui 버그, 레슨 신청 버그)과 추가 요구 기능 개발을 진행하였습니다. 그렇게 개발 완료 후, 한달이 지나자 서비스가 완전히 안정화되어 2023년 5월 이후로 버그가 발생하지 않은 채 현재까지 정상적으로 서비스가 운영되고 있습니다.

이후 2023년 8월 피아노 투게더의 시스템을 보고 저희 프로그램을 사용하고 싶다는 다른 음악 학원이 연락이 와 추가 개발을 진행하게 되었습니다. 해당 음악 학원은 음악/미술 종합학원으로 지점이 하나인 대신 과목이 많았습니다. 따라서 피아노 투게더의 경우와 다르게 지점을 없애고 과목을 만들었습니다. 또한, 기존 지점은 변경 불가 정보였지만, 과목은 관리자가 직접 조작 가능하도록 db에 추가 할 필요가 존재했습니다. 이외에도 강사의 수가 많아 ui가 깨지는 문제 등 클라이언트의 상황에 맞춰 변경사항을 적용하여 기존 프로그램을 일부 수정하여 개발하였습니다. 계약 이후 2주간의 개발 후, 클라이언트에게 전달하였고 1달간의 추가 유지보수 기간 끝에 정상 운영되고 있습니다.
